  /help/events/{eventId}/stream:
    get:
      summary: Real-time event updates (WebSocket)
      description: |
        Opens a **WebSocket** stream that delivers live updates to all
        participants of the help event.

        The client must send **no initial message**; the server begins streaming.

        ## Example Broadcast Messages:
        ### Location Visibility Rules
        - **Rescuee** receives **all** location updates from participating rescuers.
        - **Rescuer** receives **only the rescuee’s** location updates.
        - **Rescuer never receives other rescuers' locations** (privacy).

        ### Rescuer Joined
        ```json
        {
          "type": "rescuer_joined",
          "userId": "user_123",
          "acceptedAt": "2025-11-05T12:00:00Z"
        }
        ```

        ### Rescuer Withdrew
        ```json
        {
          "type": "rescuer_withdrew",
          "userId": "user_567"
        }
        ```

        ### Location Update
        ```json
        {
          "type": "location_update",
          "userId": "rescuer_42",
          "location": { "latitude": 60.19, "longitude": 24.94, "accuracy": 12.5 }
        }
        ```

        ### Event Status Change (e.g., cancelled or completed)
        ```json
        {
          "type": "event_status_changed",
          "status": "cancelled"
        }
        ```

        ### Rescuee Battery Low
        ```json
        {
          "type": "battery_level_changed",
          "userId": "rescuee_91",
          "batteryLevel": 12
        }
        ```

      parameters:
        - name: eventId
          in: path
          required: true
          schema:
            type: string
      responses:
        "101":
          description: Switching to WebSocket protocol
